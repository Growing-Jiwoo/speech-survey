## 1. 배경과 목표

기존 사이트는 "영어 문장 30개를 읽고 STT로 평가"하는 구조였다. 요구사항이 다음과 같이 바뀌었다:

1. **STT 완전 제거.** 아이가 녹음하면 녹음 파일 저장 + 완료 여부만 추적. 참여자는 자기 녹음을 재생할 수 없다.
2. **문항을 KODYS-G1 검사지로 교체.** 낱말 해독 14 + 문장 읽기유창성 4 + 낱말 쓰기 10 + 검사자 체크리스트 1 = 총 29문항.
3. **참여자 정보 확장.** 학교명(지역→학교 검색 선택)/생년월일/학년·반/성별/이름/담임교사명/담임 연락처.
4. **결과(검토) 페이지 신설.** 문항별 완료 여부 일람 + 문항 바로가기 + 제출 확인 모달.

## 3. 전체 플로우

```
시작 페이지(참여자 정보 입력, 세션 생성)
→ 마이크 확인 (기존 유지)
→ 문항 1~28 (이전/다음 내비게이션)
→ 문항 29: 검사자 체크리스트 [제출] 버튼
→ 결과(검토) 페이지: 문항별 완료 여부 + 문항 바로가기
→ [제출] → 확인 모달
   ├ "네"  → POST /api/sessions/submit (낱말쓰기 답 + 체크리스트 + submitted_at) → 종료 페이지
   └ "아니오" → 모달 닫힘
```

- 세션 행은 **시작 시 생성**한다 (녹음 업로드에 세션 ID가 필요).
- 녹음 파일은 **문항마다 즉시 업로드**한다 (새로고침·이탈에도 유실 없음).
- 낱말쓰기 예/아니오와 체크리스트는 sessionStorage에 보관했다가 **최종 "네"에서 DB 일괄 저장**한다.
- 진행 상태 전체(현재 문항, 녹음 완료 여부, 답변)는 sessionStorage에 유지 → 새로고침해도 이어서 진행.

## 6. 문항 화면 동작

**공통:** 모든 문항에 이전/다음 버튼 (1번의 '이전'은 비활성). 진행바 유지. 별도 '건너뛰기' 버튼과 skip API는 제거 — 녹음 없이 '다음'을 누르는 것이 곧 건너뛰기이며 결과 페이지에 미완료로 표시된다.

**녹음 문항 (1~18):**
- 안내 문구: 낱말 "아래 낱말을 소리 내어 읽어 주세요" / 문장 "아래 문장을 소리 내어 읽어 주세요".
- 녹음 시작 → 카운트다운(낱말 30초/문장 40초) 표시 → 수동 종료 또는 시간 만료 시 자동 종료 → 즉시 업로드.
- 업로드 성공 = 문항 완료. "목소리가 잘 담겼어요" 완료 배지 표시 (저음량 경고는 기존 로직 유지).
- 재녹음 가능 (attempt_no 증가, 모든 시도 보존). **재생 기능 없음.** STT 호출 없음.
- 업로드 실패 시 오류 안내 + 마지막 녹음으로 재시도 버튼 (기존 패턴 유지).

**낱말쓰기 문항 (19~28):**
- 상단: "학생이 아래의 낱말을 정확하게 쓸 수 있나요?" → 낱말 크게 표시 → 예/아니오 선택.
- 예/아니오 중 하나를 선택해야 '다음' 활성화 (체크 필수). 선택값은 sessionStorage에 저장.

**검사자 체크리스트 (29):**
- 5개 영역 각각 체크박스 + 관찰 지표 문구. 체크는 전부 선택 사항.
- 하단 버튼명은 "제출" — 누르면 결과(검토) 페이지로 이동.

## 7. 결과(검토) 페이지

- 섹션별 그룹으로 29문항의 상태 일람:
  - 녹음 문항: 녹음 완료 / 미녹음
  - 낱말쓰기: 선택값(예/아니오) / 미선택
  - 체크리스트: 선택한 영역 나열 (없으면 "선택 없음")
- 문항 번호는 링크 — 클릭 시 해당 문항으로 바로 이동. 해당 문항에서 돌아올 수 있는 동선("검토로 돌아가기") 제공.
- 하단: **이전** 버튼(체크리스트로) + **제출** 버튼. 미완료가 있어도 제출 가능하되 미완료 개수를 안내.
- 제출 클릭 → 모달:
  - title: "녹음이 잘 되었는지 모두 확인하셨습니까?"
  - description: "※ 녹음이 잘 되지 않았을 경우 재검사 요청이 갈 수 있습니다."
  - **네** → `POST /api/sessions/submit` 성공 시 종료 페이지로 이동. **아니오** → 모달 닫힘.

## 8. DB 스키마 (`supabase/migrations/003_kodys_redesign.sql`)

기존 `questions`/`responses`/`attempts` drop (기존 수집분은 테스트 데이터).

```sql
-- sessions: 참여자 정보 확장 (child_age 제거)
sessions (
  id uuid PK,
  school_region text not null,      -- 교육청명
  school_id text not null,          -- 원본 학교ID
  school_name text not null,
  birth_ymd char(6) not null,       -- YYMMDD
  grade int not null check (grade between 1 and 6),
  class_no int not null check (class_no between 1 and 99),
  gender text not null check (gender in ('남','여')),
  child_name text not null,
  teacher_name text not null,
  teacher_contact text not null,
  checklist text[] not null default '{}',  -- 최종 제출 시 저장 (영역 코드)
  started_at timestamptz default now(),
  submitted_at timestamptz          -- 최종 제출 시각 (null=미제출)
)

-- recordings: 녹음 즉시 기록 (모든 시도 보존)
recordings (
  id uuid PK, session_id FK,
  item_code text not null,          -- rw01~rs04
  attempt_no int not null,
  audio_path text not null,
  duration_sec numeric(5,2),
  created_at timestamptz default now(),
  unique (session_id, item_code, attempt_no)
)

-- writing_answers: 최종 제출 시 일괄 저장
writing_answers (
  id uuid PK, session_id FK,
  item_code text not null,          -- ww01~ww10
  can_write boolean not null,
  unique (session_id, item_code)
)
```

RLS 전면 차단 + service role 접근 (기존 방식 유지). Storage 버킷 `recordings` 유지, 경로 `<sessionId>/<itemCode>_<attemptNo>.<ext>`.

## 10. 관리자 개편

- **목록:** 학교 · 학년/반 · 이름 · 생년월일 · 제출 여부 · 녹음 완료 수(n/18).
- **상세(결과지):** 섹션별 표시 —
  - 녹음 문항: 문항 텍스트 + 시도별 듣기(`AudioPlayer` 재사용, signed URL) + 미녹음 표시. 의미/무의미 구분 표기.
  - 낱말쓰기: 낱말 + 예/아니오 표.
  - 체크리스트: 선택 영역.
  - 참여자 정보 헤더(학교·학년반·성별·생년월일·담임·연락처).
- STT 텍스트·자동 비교 컬럼 완전 제거. 로그인/인증 방식은 기존 유지.

## 13. 범위 밖 (명시)

- 접수된 녹음의 자동 채점·분석 (검사자가 직접 청취·채점)
- 학교 데이터의 주기적 자동 갱신 (빌드 스크립트 수동 재실행)
- 관리자 페이지의 재검사 요청 발송 기능 (모달 문구는 안내일 뿐)
