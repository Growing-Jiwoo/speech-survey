# 설문 리디자인 + 입력 검증 변경 — 디자인 스펙

**작성일:** 2026-07-13
**상태:** 사용자 확정 (v3 목업 승인 완료)
**선행 문서:** `2026-07-13-stt-survey-site-design.md` (기능 골격 — 본 스펙은 그 위의 UI/UX·검증·진행 로직 개정)
**시각 레퍼런스:** `2026-07-13-redesign-mockup-v3.html` (이 디렉토리, 브라우저로 열어 확인 — 확정된 v3 목업 원본)

## 1. 목표

1. **입력 검증 변경** — 이름·나이 제한 완화 및 문자 제한 강화.
2. **전면 리디자인** — "AI가 만든 티"(기본 이모지, 파스텔 카드, 개성 없는 폰트) 제거. 직접 창작한 캐릭터 **블립(Blip)** 기반의 차분한 **설문/검사 도구** 톤.
3. **평가 비노출 원칙** — 아이는 자신의 정오를 알 수 없어야 함. 평가는 선생님 결과지에서만.

### 비목표 (Non-goals)
- 배포, 브랜치 병합, 학생/선생님 명칭 최종 결정(미정 — UI는 중립 표기 "결과지", "이름" 사용).
- DB 스키마 변경 (자동 비교는 조회 시 계산).
- 효과음/사운드.

## 2. 입력 검증 (프론트+서버 동일 규칙)

### 이름
- **허용:** 완성형 한글(`가-힣`)과 영문(`a-zA-Z`)만. 단어 사이 단일 공백 허용. 그 외(숫자·특수문자·자모 단독·연속 공백·앞뒤 공백) 불가.
- **정규식:** `/^[가-힣a-zA-Z]+( [가-힣a-zA-Z]+)*$/`, 최대 30자.
- **IME 안전 (필수):** 한글 조합 중 키 입력마다 필터링하면 조합이 깨진다. 조합 중(`isComposing`)에는 개입하지 않고, `compositionend`·blur·제출 시점에만 검증/정리한다.
- 오류 문구(중립): "이름은 한글이나 영어로만 쓸 수 있어요."

### 나이
- **허용:** 숫자만, 1~3자리, 정수 1 이상. **기존 3~19 범위 제한 삭제.**
- 입력: `inputMode="numeric"` + 비숫자 즉시 제거(숫자는 IME 조합 문제 없음).
- 서버: `Number.isInteger(n) && n >= 1 && n <= 999`.

### 서버 미러링
`/api/sessions`의 검증을 위 규칙으로 교체. 클라이언트 검증은 UX용, 서버 검증이 최종 방어선.

## 3. 디자인 시스템 (v3 확정)

### 팔레트 (globals.css `@theme`)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--color-blue` | `#2F6BFF` | 주 색. CTA, 포커스, 진행바, EQ 막대 |
| `--color-blue-deep` | `#1E4FCC` | 버튼 아래 입술(3px), 눌림 |
| `--color-red` | `#E5484D` | 녹음 중 전용 (버튼·상태점·안테나·링) |
| `--color-red-deep` | `#C13A3E` | 녹음 버튼 입술 |
| `--color-mint` | `#0BA37A` | **결과지 전용** "일치" 배지 |
| `--color-amber` | `#B97F10` | **결과지 전용** "불일치" 배지 |
| `--color-ink` | `#0E1526` | 본문 텍스트 |
| `--color-ink-soft` | `#3A4256` / `--color-ink-mute` `#8A94AD` | 보조/힌트 |
| `--color-bg` | `#F4F6FB` (페이지) / 화면 안 `#FCFDFF` | 배경 |
| `--color-line` | `#E3E8F3` / `--color-well` `#F7F9FE` | 테두리/입력 배경 |

원칙: 아이 화면에서 빨강=녹음 중이라는 **기능 신호로만** 사용. mint/amber는 아이 화면에 등장 금지(평가색).

### 타이포그래피
- **Jua 폰트 삭제.** `next/font/google`로 **Noto Sans KR**(400/500/700) 기본 + **Lexend**(400/500/600).
- 영어 제시 문장·STT 텍스트는 Lexend(`--font-read`) 22px/500, line-height 1.45.
- 제목은 고딕 700 (디스플레이 폰트 없음 — 놀이 톤 방지).

### 캐릭터 — 블립 (components/Blip.tsx, 인라인 SVG)
- 파란 라운드 사각 몸통(그라데이션 `#5A90FF→#2454DB`)+안테나+발. 목업 SVG를 그대로 컴포넌트화.
- **역할: 로고 + 녹음 상태등.** 감정 연출(축하·수면·응원) 없음.
- variant: `logo`(브랜드 행, 30px) / `idle`(안테나 파랑) / `recording`(안테나 빨강+펄스, 입 열림).
- 눈 깜빡임(4.6s 주기)만 유지 — 생명감 최소치.

### 컴포넌트 스타일
- **카드:** 흰 배경, 1px `--color-line`, radius 18px, 그림자 `0 10px 24px -18px rgba(30,79,204,.25)`.
- **CTA 버튼:** 파랑, 54px, radius 15px, **아래 입술 3px**(`box-shadow: 0 3px 0 deep`) + 부드러운 그림자. active 시 입술 흡수(translateY). 이것이 "살짝 3D"의 전부.
- **원형 버튼(마이크/정지):** 92px, 동일 입술 구조. 정지 버튼은 빨강 + **20초 링 타이머**(SVG stroke-dashoffset, 실제 경과시간 연동. 숫자 카운트다운 금지).
- **입력:** 50px, well 배경, 포커스 시 파랑 테두리+링.
- **EQ 레벨미터:** 파란 막대 5개, recorder.level 연동 (기존 LevelMeter 재스타일).
- **진행도:** "문항 4 / 12" 텍스트 + 6px 얇은 바. 별·스텝 장식 금지.
- **모션:** 깜빡임·안테나 펄스·링·EQ만. `prefers-reduced-motion: reduce` 시 전부 정지. 이모지 전면 금지(전 화면).

## 4. 아이 플로우 (평가 비노출)

### 화면 1 — 시작 (`app/page.tsx`)
브랜드 행(블립 logo + "말하기 설문") / "안녕하세요!" / 안내 1문장 / 이름·나이 카드 / "시작하기" / 하단 고지 "녹음된 목소리는 설문 확인 용도로만 사용돼요."

### 화면 2 — 마이크 확인 (survey 페이지 mic phase)
"마이크 확인" / "버튼을 누르고 '안녕하세요'라고 말해 주세요." / 마이크 버튼(리플) / EQ / peak 감지 성공 시 하단 "설문 시작" 활성화. 게임 프레이밍 금지. 실패 시: "목소리가 잘 안 들려요. 마이크 가까이에서 다시 한번 해 주세요." (기술 안내)

### 화면 3 — 문항·녹음 중
진행도 / 문항 카드("아래 문장을 소리 내어 읽어 주세요" + Lexend 문장 — **읽어주기(TTS) 버튼 없음, 검사 타당도**) / 블립 recording + 빨간 점 "녹음 중" + EQ / 하단 정지 버튼(링 타이머) + "다 읽었으면 버튼을 눌러 주세요" / "이 문장 건너뛰기"(inline 확인: "정말 건너뛸까요? 네/아니요").

### 화면 4 — 문항·녹음 후
동일 문항 카드 / 확인 행 "✓ 목소리가 잘 담겼어요." / **"다음 문장"** CTA / 아래 작은 링크 **"다시 녹음하기"** (유지 확정 — attempt_no 증가, 모든 시도 저장).
- **STT 결과·재생 플레이어·정오 표시 절대 노출 금지.**
- **무음 감지(기술 체크):** `rec.peak < 0.01`이면 업로드는 하되 확인 행 대신 "목소리가 잘 안 담긴 것 같아요. 한 번 더 말해 볼까요?" 표시. "다음 문장"은 **막지 않는다**(가짜 양성으로 진행이 막히면 안 됨).
- 업로드 실패(네트워크): "저장에 문제가 생겼어요. 다시 시도해 주세요." + 재시도 버튼 (기존 lastRec 재전송 로직 유지).

### 화면 5 — 완료 (`app/done/page.tsx`)
블립 idle / "설문이 끝났어요" / "참여해 줘서 고마워요. 이제 화면을 선생님께 보여 주세요." / "처음 화면으로". 축하 연출 금지.

### 진행 로직 변경 (핵심)
- 기존: STT 텍스트 비어있지 않아야 succeeded 래치 → 다음 활성화. **폐기.**
- 신규: **녹음 업로드(응답 저장) 성공 = 다음 활성화.** STT가 비거나 실패해도 오디오는 저장되고 진행 가능 — 판단은 선생님이.
- `/api/transcribe` 응답에서 **sttText 제거** (`{ ok: true }` 수준만) — 아이 기기로 인식 결과가 전송조차 되지 않게(개발자도구 노출 차단). STT 수행·DB 저장은 서버에서 기존대로.

## 5. 선생님 화면 (관리자)

톤: 차분한 데이터 중심(아이 화면과 의도적 분리). 블립은 로고로만.

### 목록 (`app/admin/page.tsx`)
상단 바(로고+"말하기 설문 · 관리자") + KPI 칩(세션 수·완료율·오늘) + CSV 버튼(ink 배경). 테이블: 이름/나이/시작/진행(n/12)/상태 pill(완료 mint·진행 중 amber)/건너뜀.

### 결과지 (`app/admin/[id]/page.tsx`)
- 헤더: "결과지 — {이름} ({나이}세)" + KPI 칩(완료 n/12 · 자동 일치 n · 건너뜀 n).
- 문항 테이블: # / 제시 문장 / 들린 말(STT) / **자동 비교 pill** / 듣기(원형 재생 버튼, 기존 서명 URL 오디오).
- **자동 비교(신규, `lib/compare.ts`):** 정규화(소문자화·구두점 제거·공백 축약) 후 완전 일치 여부.
  - `일치`(mint) / `불일치`(amber) / `인식 안 됨`(회색, STT 빈 값) / `건너뜀`(회색).
  - 재시도 다건이면 **모든 시도 표시**(시도 번호 포함), 비교는 각 시도별.
- 하단 고지: "자동 비교는 참고용입니다 — 최종 평가는 녹음을 직접 듣고 판단해 주세요."
- CSV에 자동 비교 컬럼 추가 (`match`: matched/mismatched/unrecognized/skipped).

### 로그인 (`app/admin/login/page.tsx`)
같은 차분한 톤으로 재스타일 (로직 변경 없음).

## 7. 완료 기준

1. 이름 "김 지우", "Mary Jane", "지우Kim" 허용 / "지우1", "지우!" 거부. 나이 1~999 허용, "20" 허용(기존 거부됨), 비숫자 거부.
2. 아이 플로우 어디에도 STT 텍스트·정오·칭찬·별·이모지 없음. 네트워크 탭에도 sttText 미노출.
3. 녹음 업로드 성공만으로 다음 진행 가능(STT 빈 값 포함).
4. 결과지에서 문항별 STT·자동 비교·오디오 청취 가능, CSV에 match 포함.
5. 기존 테스트 + 신규 검증 테스트 통과, typecheck clean, build 성공.
6. `prefers-reduced-motion`에서 애니메이션 정지.
