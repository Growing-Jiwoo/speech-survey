# 데스크톱 UI 설계 (2026-07-17)

## 배경 · 문제

참여자(아동) 화면 전부가 `max-w-md`(448px) 모바일 컬럼으로 고정돼 있어, 데스크톱에서는
넓은 화면 가운데 좁은 모바일 UI가 떠 있는 형태다. 관리자 화면은 이미 전폭으로 개편됐다.

학교 현장의 실제 기기 시나리오:
- **태블릿**(주 시나리오): 아동이 들고 검사 — 현행 모바일 UI가 최적. **바꾸지 않는다.**
- **컴퓨터실 데스크톱/교사 노트북**(이번 대상): 아동이 모니터 앞에 앉아 헤드셋 마이크로 검사,
  또는 교사가 노트북으로 정보 입력·검토. 모니터는 태블릿보다 시청 거리가 멀다(30cm → 60cm+).

## 설계 원칙 (도메인 제약이 곧 디자인 근거)

1. **상호작용은 동일, 무대만 확장한다.** 검사 도구는 기기 간 결과 비교 가능성(검사 일관성)이
   생명이다. 버튼 배치·흐름·단계는 어떤 기기에서도 같아야 하고, 달라지는 것은 "보이는 크기"뿐.
2. **시청 거리를 보정하는 타이포 스케일.** 모니터는 시청 거리가 태블릿의 약 2배 —
   제시어를 그에 비례해 키워야 아동 눈에 닿는 시각 크기(visual angle)가 유지된다.
   즉 데스크톱에서 글자를 키우는 것은 "변경"이 아니라 검사 경험의 **보존**이다.
3. **검사 화면은 슬라이드다(시그니처).** 데스크톱에서 검사 화면을 수직 중앙 정렬해
   "수업 시간에 들어 보이는 낱말 카드"처럼 무대화한다. 모바일의 `mt-auto`(버튼 화면 하단 고정)는
   1080p에서 콘텐츠와 버튼 사이가 텅 비는 문제가 있는데, 중앙 정렬 블록이 이를 함께 해결한다.
4. **교사 화면은 밀도, 아동 화면은 크기.** 시작 폼(교사 입력)은 2열 그리드로 스크롤을 줄이고,
   검사 화면(아동)은 요소 수를 늘리지 않고 크기만 키운다.
5. **장식은 더하지 않는다.** 읽기 검사 중 배경 장식은 아동 주의를 뺏는다. 기존 토큰
   (색·폰트·카드·버튼)을 그대로 쓰고, 데스크톱 전용 신규 시각 요소는 만들지 않는다.

## 브레이크포인트 전략

| 구간 | 폭 | 취급 |
|---|---|---|
| base~sm | <768px | **현행 유지** (폰·태블릿 세로 = 주 검사 기기) |
| md | 768~1023px | 완만한 확장(컨테이너 소폭 확대) — 태블릿 가로 |
| lg+ | ≥1024px | 데스크톱 레이아웃(그리드·타이포 스케일·수직 중앙 정렬) |

JS 기기 감지 없이 Tailwind 반응형 프리픽스만 사용(SSR 안전, 창 크기 변화 대응).

## 화면별 설계

### `/` 시작 화면 — 사용자: 교사 (데이터 입력 효율)
- 컨테이너: `max-w-md` → `lg:max-w-2xl`(672px)
- 폼 내부를 `lg:grid-cols-2`로: 학교명(전폭) → 생년월일(전폭, 3셀렉트 여유) →
  [학년·반 | 성별] → [이름 | 담임교사명] → 담임 연락처(전폭) → 동의 카드(전폭) → 시작하기
- 첫 에러 필드 포커스(`data-field`)·검증 로직은 그대로(마크업 래퍼만 그리드 셀로)

### 마이크 확인 + `/survey` 검사 화면 — 사용자: 아동 (+교사 관찰)
- 컨테이너: `lg:max-w-2xl`, **lg에서 수직 중앙 정렬**(justify-center), 내비 버튼은
  `mt-auto` → `lg:mt-12`(콘텐츠를 따라오게)
- 제시어 스케일: 낱말 38px → `lg:text-[64px]`, 문장 22px → `lg:text-[30px]`
- 낱말 쓰기 제시어도 동일 스케일, 예/아니오·내비 버튼은 크기 유지(폭만 자연 확장)
- 진행 표시·이름 표시·상태 슬롯 구조 변화 없음

### `/review` 검토 — 사용자: 교사 (한눈 훑기)
- 컨테이너: `lg:max-w-4xl`, 섹션 카드 4장을 `lg:grid-cols-2`로 — 스크롤 1화면 내 검토
- 하단 버튼은 그리드 밖 전폭 유지

### `/done` — 공용
- 중앙 정렬 유지, `lg`에서 타이포만 소폭 확대

### 관리자 — 변경 없음 (이미 전폭)

## 검증 계획
- 375(모바일)·768(태블릿)·1280·1600(데스크톱) 뷰포트에서 스크린샷 비교
- 모바일 뷰가 현행과 픽셀 수준으로 동일한지(회귀 없음) 우선 확인
- lint / typecheck / test / build

## 제외(비목표)
- 검사 흐름·버튼 배치 변경, 데스크톱 전용 기능(단축키 등), 다크 모드, 신규 시각 장식
